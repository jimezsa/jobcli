# Tailor LaTeX CV To Job

Goal: create a job-specific `.tex` copy and regenerated PDF that align truthfully with a target job posting, preserve the original CV design, and pass basic ATS parseability checks. The source `.tex` is read-only.

## Required Inputs

- Source `.tex` CV path.
- Job position link or pasted job description.
- Optional output root. Default to `tailored-cvs/` beside the source CV or in the current project workspace.
- Optional constraints, such as one page, English/Spanish, ATS-first, preserve photo/header, or avoid changing section order.

If the source CV was not already created, use `skills/pdf-cv-to-latex` first when the user starts from a PDF CV.

## Required Tools

- `pdftotext` from Poppler for ATS text extraction.
- A LaTeX engine available on `PATH`: `pdflatex`, `xelatex`, or `lualatex`.
- Python 3 for bundled helper scripts.

Use these scripts when helpful:

- `scripts/create_cv_workspace.py`: create a derived workspace and copy local LaTeX assets.
- `scripts/ats_text_check.py`: extract generated PDF text and compare it with the job description.

## Workflow

### 1. Capture The Job Description

Fetch the job link when possible. If the page requires login, blocks scraping, hides content behind JavaScript, or returns incomplete text, ask the user to paste the job description.

Save the cleaned job text in the derived workspace as `job-description.txt`. Extract:

- target title, company, location, work mode, and seniority
- required skills and tools
- preferred skills and domain terms
- responsibilities and repeated keywords
- education, certification, language, or authorization constraints

### 2. Create A Derived Workspace

Never edit the source `.tex` in place. Create a workspace before making changes:

```bash
python3 skills/tailor-latex-cv-to-job/scripts/create_cv_workspace.py \
  path/to/source-cv.tex \
  --job-label "company role" \
  --out-root tailored-cvs
```

Work only on the derived `.tex` reported by the script. Keep copied assets, generated PDFs, ATS reports, and LaTeX auxiliary files inside that workspace.

### 3. Analyze Evidence Before Editing

Read the original CV and job description. Build a private alignment map:

- job requirement
- matching evidence already present in the CV
- proposed CV edit
- gap if the CV does not support the requirement

Only tailor from existing evidence. Do not invent skills, dates, employers, titles, degrees, certifications, metrics, publications, languages, locations, or authorization status.

For detailed rules, read `references/tailoring-policy.md`.

### 4. Tailor The Derived `.tex`

Preserve the existing LaTeX structure and visual style from `pdf-cv-to-latex` unless the user asks for an ATS-first redesign.

Prefer targeted edits:

- rewrite the profile or summary to reflect the target role
- reorder skills so required tools appear early when truthfully present
- rephrase bullets with job-language keywords supported by the original CV
- move the most relevant projects or experience higher when the format allows it
- add common acronym expansions when both forms are truthful, such as `CI/CD` and `continuous integration`
- remove or compress low-relevance bullets only when needed for length

Avoid broad rewrites. Keep contact details, dates, company names, titles, and education unchanged unless the user explicitly provides corrections.

### 5. Compile The Tailored PDF

Compile only the derived `.tex`:

```bash
cd tailored-cvs/<workspace>
pdflatex -interaction=nonstopmode cv-tailored.tex
```

Run the compiler twice if `hyperref`, labels, or references require it. If the source CV uses `fontspec`, custom system fonts, or Unicode math, use `xelatex` or `lualatex` instead:

```bash
xelatex -interaction=nonstopmode cv-tailored.tex
```

Fix LaTeX errors in the derived workspace only. If the source uses local image assets, confirm they were copied or that relative paths still resolve.

### 6. Run ATS Validation

Run the ATS check against the generated PDF and job text:

```bash
python3 skills/tailor-latex-cv-to-job/scripts/ats_text_check.py \
  --pdf tailored-cvs/<workspace>/cv-tailored.pdf \
  --job-text tailored-cvs/<workspace>/job-description.txt \
  --out tailored-cvs/<workspace>/ats-report.md \
  --format md
```

Inspect:

- whether the PDF text is selectable and extractable
- whether standard sections are detected
- whether contact info is text-readable
- which job keywords are present or missing
- which missing terms are honest gaps rather than fixable wording issues

Revise, recompile, and rerun the check when missing terms are supported by the CV but absent from the tailored PDF. Do not chase a perfect score by adding unsupported or hidden terms.

### 7. Return Artifacts

Return:

- path to the tailored `.tex`
- path to the tailored PDF
- path to the ATS report
- concise change summary
- remaining gaps that cannot be fixed truthfully
- confirmation that the original `.tex` was not changed

## Done Criteria

- The original `.tex` has not been modified.
- The tailored `.tex` compiles successfully.
- The generated PDF has extractable text.
- The ATS report has no critical parseability warnings.
- Required job terms that are truthfully supported by the CV are present in natural resume language.
- Unsupported job requirements are reported as gaps, not fabricated into the CV.
